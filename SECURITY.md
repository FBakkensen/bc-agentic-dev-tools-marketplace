# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please send an email or contact the maintainers directly through GitHub.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- We will acknowledge receipt within 48 hours
- We will provide an initial assessment within 7 days
- We will work with you to understand and resolve the issue

## Scope

This security policy applies to:

- All PowerShell scripts in this repository
- Configuration files and templates
- Documentation that could lead to security issues

## Best Practices for Users

- Always review scripts before running them
- Keep PowerShell and dependencies up to date
- Use appropriate permissions when running scripts
- Do not store secrets in configuration files
